# Concept Vessel Public Rights Notice v0.1

**Date:** 2026-04-14
**Status:** Draft for owner and counsel review
**Intended use:** Public-facing legal notice for the Hull 20098 / concept-vessel showcase repo

## 1. Naming And Scope

This document is written for the public showcase repo associated with the lead concept vessel currently identified as **Hull 20098**. If the vessel is later presented under a settled public name, the notice can be renamed without changing the legal boundary it describes.

## 2. What This Repository Is

This repository is a **public showcase surface** for the concept vessel and selected public-facing outputs associated with it.

It may contain:

- public descriptions of the vessel concept
- renders, diagrams, and curated visuals
- selected output artifacts
- selected geometry views
- selected properties, dimensions, and result summaries
- progress updates and public programme notes

Some public artifacts may be preliminary, selective, or high-level. Their presence in this repository does not mean they form a complete design package or a final production specification.

This repository is **not**:

- a code release
- an unrestricted design-source release
- a class-approved ship package
- a build-ready or manufacture-ready package

No licence is implied by public posting, public display of outputs, or public availability of this repository on GitHub.

Public display of vessel materials here does **not** grant any right to:

- build, manufacture, reproduce, or commercially exploit this vessel design
- commercialize the vessel design package
- treat the public materials as a complete engineering package
- redistribute deeper package materials as if they were publicly licensed

## 4. GitHub Public-Repository Reality

Because this repository is public on GitHub, GitHub and GitHub users receive limited service-level permissions under GitHub's Terms of Service. In practice, that means public GitHub visibility allows others to view the repository and use GitHub-native functionality such as forking within the GitHub service.

Those limited permissions arise from GitHub's service terms and public-repo functionality. They are **not** a separate vessel-design, manufacturing, or commercial licence granted by this notice.

That public-service visibility does **not** mean:

- a broader design licence has been granted
- the vessel can be built from public repo contents
- the public repo is an open-hardware release
- commercial manufacture or derivative-vessel rights are available

If a right matters commercially or operationally, assume it is **not granted** unless it appears in a separate written agreement.

## 5. Selected Outputs Are Public Showcase Material, Not A Full Package

The repository may show selected outputs and latest results on an ongoing basis. That is intentional.

But public showcase of selected outputs does **not** convert the repository into:

- a full technical diligence pack
- a final ship-design package
- a yard-ready or class-ready package
- a grant of engineering reliance rights
- a design, build, derivative-development, or manufacture licence

The public repo should therefore be treated as a showcase and signalling surface, not as the full controlled package for deeper commercial or technical use.
